# Research Radar 使用指南

网址：**https://treeEnergy.github.io/research-radar/**

---

## 一、网站是什么

Research Radar 是一个专为你的研究方向（压气机稳定性、叶尖间隙、机匣处理、进气畸变、稳定性建模）定制的文献追踪网站。它会自动从四本核心期刊抓取论文，用 AI 分析并翻译摘要，让你快速了解领域最新进展。

**四本目标期刊：**
- Journal of Turbomachinery
- Journal of Engineering for Gas Turbines and Power
- Journal of Propulsion and Power
- AIAA Journal

---

## 二、日常使用

### 浏览和搜索论文

打开网站，默认显示所有论文，按相关度排序。

**搜索：** 顶部搜索框支持中英文，会同时匹配标题、摘要、中文综述。

**筛选标签：** 搜索框下方有标签栏，点击可按分类筛选（稳定性分析、叶尖间隙、机匣处理等）。

**排序：** 右侧下拉可切换"相关度""日期""期刊"排序。

**已读过滤：** 可筛选"仅未读"或"仅已读"。

**相关度颜色含义：**
| 颜色 | 分数 | 含义 |
|------|------|------|
| 红色 | 5 分 | 核心方向 |
| 黄色 | 4 分 | 高度相关 |
| 绿色 | 3 分 | 相关 |
| 蓝色 | 2 分 | 部分相关 |

### 查看论文详情

点击论文标题或"展开"按钮，可看到：
- AI 生成的中文综述
- 研究方法、创新点、主要结论
- 原文链接

### 标记已读

每篇论文右下角有"标记已读"按钮，点击后论文变为已读状态（背景色变化），再次点击可取消。已读状态保存在浏览器本地，刷新不丢失。

### 写评价/笔记

点击论文卡片上的"写评价"按钮，可以输入你对这篇论文的备注。保存后按钮变为"编辑评价"，表示已有笔记。笔记保存在浏览器本地。

## 三、课题组追踪

点击顶部导航的"课题组"Tab，可以看到系统预设的 6 个课题组（MIT Greitzer、Purdue Key、NASA Hah 等），以及你自定义的课题组。

**筛选某课题组的论文：** 点击课题组卡片，论文列表自动筛选为该课题组成员发表的论文。

**自定义课题组：** 点击课题组卡片上的编辑按钮（✏️），可以修改名称、机构、PI 姓名列表。PI 姓名用逗号分隔，系统会自动匹配论文作者字段。

---

## 四、AI 对话

点击右下角聊天气泡图标打开对话窗口。

**首次使用：** 会弹窗要求输入 DeepSeek API Key（`sk-` 开头），填入后自动保存，以后无需再填。

**能问什么：**
- "机匣处理对叶尖泄漏涡有什么影响？"
- "Moore-Greitzer 模型的最新进展有哪些？"
- "列出近两年关于进气畸变的论文"
- "Greitzer 课题组最近在研究什么？"

AI 会优先检索你的本地论文库回答，并注明引用了哪些论文。如果库里没有相关内容会明确告知。

**注意：** AI 对话会消耗 DeepSeek 额度，建议充值 50-100 元，够用很久。

---

## 五、更新论文库

点击右上角 **▶ 更新检索** 按钮。

**首次使用：** 会弹窗要求输入 GitHub Token（`github_pat_` 开头），填入后自动保存。

点击后会触发 GitHub Actions 自动运行流水线：
1. 从 OpenAlex 按关键词抓取最新论文
2. 用 DeepSeek AI 分析每篇论文
3. 自动保存到仓库，网站自动更新

整个流程约需 **10-20 分钟**。面板会显示"排队中→运行中→完成"。

**完成后：** 点面板里的"刷新数据"按钮，论文列表即时更新，无需重载页面。

**自动更新：** 系统每周一凌晨 2 点（UTC）自动运行一次，无需手动触发。

## 六、修改检索关键词/期刊

如果想添加新的关键词或追踪新课题组，需要修改配置文件。

1. 打开本地项目文件夹 `E:\claude-project\20260327\scripts\config.py`
2. 在 `KEYWORDS` 列表中添加新关键词（英文，加引号）
3. 在 `RESEARCH_GROUPS` 列表中添加新课题组
4. 保存后，在终端执行：

```bash
cd E:/claude-project/20260327
git add scripts/config.py
git commit -m "config: add new keywords"
git push
```

推送后，下次流水线运行时会用新配置抓取。

## 七、常见问题

**Q：网站打开空白或论文不显示？**
等 1-2 分钟，GitHub Pages 部署需要时间。或者强制刷新（Ctrl+Shift+R）。

**Q：AI 对话报错"API Key 无效"？**
打开浏览器开发者工具（F12）→ Console，输入：
```js
localStorage.removeItem('rr_deepseek_key')
```
然后重新填入正确的 Key。

**Q：触发流水线报错"Token 无效"？**
同上，在 Console 输入：
```js
localStorage.removeItem('rr_github_token')
```
然后重新填入 GitHub Token。

**Q：标注/已读记录丢了？**
标注存在浏览器本地（localStorage）。换了浏览器、清除浏览器数据、或换了电脑会丢失。建议固定用一个浏览器访问。

**Q：如何查看流水线详细日志？**
点击流水线面板里的链接（`https://github.com/treeEnergy/research-radar/actions`），可以看到 GitHub Actions 的完整运行日志。

**Q：论文库上限是多少？**
最多保留 500 篇，按日期倒序，超出自动截断。
