# Timeline + Wishlist Design Spec

**Date:** 2026-03-29
**Status:** Approved

---

## Goal

Add two independent features to Research Radar:

1. **Timeline view** — A heatmap (topic × decade) showing paper counts from 1970 to present, with AI-generated era summaries in a slide panel on cell click.
2. **Wishlist** — A floating panel for recording papers the user wants to add in future pipeline runs.

---

## Feature 1+2: Timeline

### Data Layer

#### Historical Papers (`data/papers-historical.json`)

Stores all papers from 1970–2019 fetched from OpenAlex. Lightweight format — no DeepSeek AI analysis initially (to control cost), but includes all fields OpenAlex provides so AI analysis can be added later without schema changes.

```json
[
  {
    "id": "W2964203751",
    "title": "Effects of casing treatment on compressor stability",
    "date": "1982-01-01",
    "year": 1982,
    "authors": ["Smith, J.R.", "Jones, A.B."],
    "venue": "Journal of Turbomachinery",
    "source": "Journal of Turbomachinery",
    "url": "https://doi.org/10.1115/...",
    "abstract": "This paper investigates...",
    "keywords": ["casing treatment", "compressor stall"],
    "topics_matched": ["机匣处理", "压缩机稳定性"]
  }
]
```

- `topics_matched`: computed at fetch time — which of the user's topic terms match title/abstract/keywords (same `matchesTopic` logic)
- `abstract`, `keywords`: populated directly from OpenAlex, no additional cost
- Future fields (to be added later): `summary_zh`, `relevance`, `tags` (from DeepSeek analysis)
- No max-count limit (unlike `papers.json`'s 500-paper cap)

#### Timeline Precomputed Data (`data/timeline.json`)

Generated by `scripts/build_timeline.py` at the end of each pipeline run. Stores counts and AI summaries per topic × decade.

```json
{
  "generated_at": "2026-03-29T00:00:00Z",
  "decades": [1970, 1980, 1990, 2000, 2010, 2020],
  "topics": [
    {
      "label": "机匣处理",
      "decades": {
        "1980": {
          "count": 17,
          "paper_ids": ["W2964203751", "W1234567890"],
          "summary": "1980年代机匣处理研究进入爆发期。NASA和MIT相继发布系统性测量结果，确立了周向槽和轴向槽的扩稳机制基础理论……"
        },
        "1990": {
          "count": 23,
          "paper_ids": [...],
          "summary": "..."
        }
      }
    }
  ]
}
```

- Decades with zero papers have no entry (omitted)
- `paper_ids` references IDs in `papers-historical.json` (1970–2019) OR `papers.json` (2020+)
- Summaries generated by DeepSeek: input = paper titles from that cell (~10–20 titles), output = 3–5 sentence Chinese summary of research progress
- Estimated ~30 non-empty cells total → < $0.05 DeepSeek cost per timeline rebuild

#### Config Changes (`scripts/config.py`)

```python
FETCH_FROM_YEAR = 2020            # existing — current papers pipeline
FETCH_FROM_YEAR_HISTORICAL = 1970 # new — historical papers pipeline
```

### New Scripts

#### `scripts/fetch_papers_historical.py`

Fetches papers from `FETCH_FROM_YEAR_HISTORICAL` to `FETCH_FROM_YEAR - 1` using the same OpenAlex API. Key differences from `fetch_papers.py`:

- Does **not** call DeepSeek — stores raw OpenAlex fields only
- Computes `topics_matched` via keyword matching (same `matchesTopic` logic ported to Python)
- Outputs to `data/papers-historical.json`
- Incremental: skips paper IDs already in the file
- Uses same `TARGET_JOURNALS`, same keywords from `config.py` or `data/topics.json`
- Lower concurrency (4 threads instead of 8) to avoid OpenAlex rate limits on large date ranges

#### `scripts/build_timeline.py`

Reads `papers-historical.json` + `papers.json`, groups by topic × decade, generates AI summaries via DeepSeek, writes `data/timeline.json`.

Logic:
1. Load all papers from both files
2. For each paper, determine its decade (`year // 10 * 10`)
3. For each topic in `getTopics()` logic (reads `data/topics.json` if exists, else `config.py` CATEGORIES), find matching papers per decade using `topics_matched` field (historical) or `tags` field (current)
4. For each non-empty (topic, decade) cell: if no existing summary in current `timeline.json`, call DeepSeek to generate summary
5. Write updated `timeline.json`

DeepSeek prompt for era summary:
```
以下是{decade}年代关于"{topic_label}"的{count}篇论文标题：

{titles_list}

请用3-5句话（中文）概括这个年代该领域的主要研究进展和贡献。要求：具体提及关键工作或发现，避免泛泛而谈。
```

#### `run_pipeline.py` Integration

Add at the end of `run_papers_pipeline()`:
```python
# 6. 历史论文抓取（增量，仅首次或手动触发时全量）
fetch_papers_historical_incremental()

# 7. 生成时间轴数据
build_timeline()
```

Also add to GitHub Actions workflow step to commit `data/papers-historical.json` and `data/timeline.json`.

### UI — Timeline Tab

#### Navigation

Add `TIMELINE` tab to the existing nav bar, between `REPOS` and `AI CHAT`:

```
论文 | GROUPS | REPOS | TIMELINE | AI CHAT | MANUAL | BUILD
```

#### Main View: Heatmap

- Rows: current topics (from `ls_topics` or `data/timeline.json`)
- Columns: decades from 1970s to 2020s (6 columns)
- Cell content: paper count number
- Cell color: opacity scaled to count within each row (max opacity = darkest decade for that topic)
- Each topic has its own color (matches existing topic color scheme)
- Empty cells (count = 0): shown as dim with "—"
- Row header: topic label
- Column header: decade (1970s, 1980s, …)
- Below heatmap: small legend showing color scale

#### Drill-Down: Slide Panel (D1)

Triggered by clicking any non-empty cell. Panel slides from the right (same pattern as `topics-panel` and `pipeline-panel`).

Panel contents:
- Header: `{topic_label} · {decade}s` + close button
- AI summary block (gold left border, from `timeline.json`)
- Paper list: title + year + first author, sorted by year
  - Papers from `papers.json` (2020+): clickable, clicking scrolls to that paper in the 论文 tab
  - Papers from `papers-historical.json` (1970–2019): title links to DOI/URL in new tab
- If cell has > 20 papers, show first 20 with "显示全部 N 篇" toggle

#### Loading State

On first load of TIMELINE tab:
1. Fetch `data/timeline.json` and `data/papers-historical.json`
2. Show loading spinner while fetching
3. If `timeline.json` missing: show "尚未生成时间轴数据，请先运行流水线"

## Feature 3: Wishlist

### Storage

`localStorage` key: `ls_wishlist`

```json
[
  {
    "id": "uuid-or-timestamp",
    "title": "Cumpsty 1977 — Annular cascade measurements",
    "note": "经典机匣处理实验，想确认是否已在 OpenAlex 收录",
    "added_at": "2026-03-29T10:00:00Z"
  }
]
```

### UI

- Floating button bottom-right: `📋 愿望清单` (always visible regardless of active tab)
- Click → slide panel from right (same drawer pattern)
- Panel contents:
  - Header: "文献愿望清单"
  - Add form: text input (论文标题/描述, required) + textarea (备注, optional) + "添加" button
  - List of entries: title + note + date, each with delete button (✕)
  - Empty state: "暂无记录，把想找的文献记在这里"
- No AI, no backend, pure localStorage

## Out of Scope (Future)

- DeepSeek AI analysis for historical papers (structure supports it, not implemented now)
- Exporting wishlist to a file or sharing it
- Searching/filtering within the wishlist
- Annotation/read-status on historical papers
